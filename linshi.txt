// README.md 游戏策划书

# 综述

游戏名称：蚀岛。
玩法人数：单人、文字。
游戏类型：文字冒险、生存、克苏鲁。
游戏方式：H5页面。
游戏框架：ts+vue3。
主要面向手机用户。

# 游戏机制

## 核心机制

玩家通过阅读文字，了解游戏的背景、环境、人物、任务等，通过点击可交互按钮以做出操作。

## 玩家状态

玩家的属性分为生存属性、基础属性、技能。
生存属性是指与玩家生存直接相关的属性，包括生命值、饱食度、 体力值、SAN、温暖度、负重。  
基础属性是指玩家的基本属性，例如力量、敏捷、智力、体质。  
武器熟练度是指玩家使用武器的熟练程度，例如使用剑、使用弓箭等。
技能属性是指玩家的各种技能等级，包括生存技能、战斗技能。

### 生存属性

生存属性与生存息息相关。
其中生命值、饱食度、体力值、SAN均存储为整数。
若某次交互变动值小于1，则不更新该值，而是将此次交互变动值累计到下次变动直到大于1时再更新。例如第二次交互累计值为2.3，则更新2，累计值置为0.3。
生存属性当前值的变动，均含以下方式：

- 事件
- 食物
- 道具

#### 生命值

1、作用

- 生命值是指玩家当前的健康状态，生命值为0时，玩家死亡，进入结局判定。
- 额外情况：满足特定条件时，触发事件，事件判定优先于结局判定。参见[事件](#事件)。

2、上限

- 生命值上限与体质正相关。计算方式为：生命值 = 体质 \* 10，随体质变动而变动。无例外情况。
- 生命上限变动时，当前生命值按照比例变动。当前生命值不会超过上限，且不小于0。

3、损失

- 战斗中受到敌人攻击，损失生命值，具体公式参见[战斗机制](#战斗机制)。
- 饱食度、口渴度为0、温暖度不在适宜范围内时，玩家会随时间损失生命值，以分钟为单位计算。不同的相互叠加。
  - 饥饿状态，生命值每小时损失量 = 5% \* 最大生命值。
  - 温暖度不在适宜范围内时：
    - 处于寒冷、炎热环境，生命值每小时损失量 = 5% \* 最大生命值。
    - 处于极寒、酷热环境，生命值每小时损失量 = 15% \* 最大生命值。
- 受异常状态影响，玩家会随时间损失生命值，具体公式参见[异常状态](#异常状态)。
- 受事件影响，参见[事件](#事件)。

4、恢复

- 随时间生命值会自然恢复，参见5、自然恢复。
- 通过休息/睡觉恢复生命值，具体公式参见[休息](#休息)。

5、自然恢复

- 生命值会随时间自然恢复，恢复速率与体质与buff相关。
- 自然恢复每小时计算，向上取整。
- 基础恢复速率为：生命值每小时恢复量 = 体质 \* 0.1， 随体质变动而变动。
- 实际恢复速率 = 基础恢复速率 \* 恢复速率系数。恢复速率系数为1时，基础恢复速率为实际恢复速率。恢复速率系数为0时，实际恢复速率为0。
- 恢复速率系数，事件、异常状态、人数技能等会改变恢复速率系数。具体参见对应机制。

#### 饱食度

1、作用

- 饱食度是指玩家当前的饱食度状态，饱食度为0时，玩家会随时间损失生命值。

2、上限

- 饱食度上限=100\*饱食度上限系数，上限系数受事件、异常状态、人数技能影响。
- 当前饱食度不大于饱食度上限，不小于0。

3、变动

- 饱食度随时间自然损失。

4、自然损失

- 饱食度自然损失每小时损失量 = 5 \* 饱食度损失系数，以分钟为单位计算。
- 事件、异常状态、人数技能等会改变损失系数。具体参见对应机制。

5、归零

- 饱食度归零后，玩家进入饥饿状态。

#### 体力值

- 体力值是指玩家当前的体力状态，体力值小于交互所需消耗量时，无法进行此交互。
- 体力值取值范围为0-100。

1、上限

- 体力值上限=100+体力修正值，体力修正值受装备、道具、状态、事件等影响。
- 当前体力值不大于体力值上限，不小于0。

2、变动

- 体力值随时间自然恢复。
- 体力值随交互消耗。

3、自然恢复

- 体力值自然恢复每小时/回合恢复量 = 10 \* 体力恢复系数+体力恢复修正，以分钟为单位计算。
- 事件、异常状态、人数技能等会改变恢复系数。具体参见对应机制。
- 体力恢复系数默认=1，体力恢复修正默认=0

4、消耗

- 体力值消耗量 = 交互所需消耗量 \* 体力消耗系数，体力消耗系数受装备、道具、状态、事件等影响。
- 事件、异常状态、人数技能等会改变消耗系数。具体参见对应机制。
- 体力消耗系数默认=1

#### SAN

1、作用

- SAN值即玩家理智值，与游戏的核心机制相关。
- SAN值的变动会导致UI不同、同样场景同样物品同样操作的描述不同。

2、设定
SAN值即为主角理智程度，在窥探到宇宙不可名状之真相、不能理解之物、大脑无法承受之事时，会损失理智值。而舒适的环境、温暖的食物、回忆中的美好事件等，会恢复理智值。

SAN值越高，主角眼中的世界就越真实（越接近正常人类的世界），越SAN值越低，主角眼中的世界就越危险混乱，充满不可预测无法名状之物。

游戏中大量的文本、选项、对话、事件等，都以玩家的SAN值为条件进行判断。SAN值管理将是游戏的核心机制。

在本作世界观观中，亦可作为模因污染程度。

2、上限

- SAN上限=100+修正指数，修正指数受事件、异常状态、人数技能影响。
- 当前理智值不大于理智值上限，不小于0。

3、变动

- 部分场景中，理智值随时间变化。具体损失值参见场景。
- 与部分敌人战斗，理智值随回合损失。具体损失值参见敌人。
- 异常状态影响。
- 温暖度、饱食度影响。
  - 温暖度不在适宜范围内时，玩家会随时间损失SAN，每小时损失量=1，按小时计。
  - 饱食度为0时，玩家会随时间损失SAN，每小时损失量=1，按小时计。

4、设定
按照SAN值的高低，分为5个档位，具体影响按具体机制中描述进行。

- 理性：>81
  - 玩家处于理智状态，不会遇到任何梦境怪物，文字描述正常，UI效果正常。
- 不安：61-80
  - 玩家处于不安状态，极低概率遭遇梦境怪物，文字描述基本正常，出现极少量异常（涂黑、错位、混乱），UI效果轻微变形。
- 动摇：41-60
  - 玩家处于动摇状态，概率遭遇梦境怪物，文字描述少量出现异常，UI效果明显变形。
- 崩溃：21-40
  - 玩家处于崩溃状态，大概率遭遇梦境怪物，文字描述出现中等幅度异常，UI效果混乱。
- 疯狂：1-20
  - 玩家处于疯狂状态，敌人全部替换为梦境怪物，文字描述出现大量异常，替换为疯狂UI，无法正常对话。

5、归零

-玩家理智值归零后，游戏结束，进行结局判定。

#### 温暖度

1、作用
温暖度是指玩家当前的温暖度状态，受天气、季节、装备、道具、环境影响。
不在适宜范围内，将持续扣除HP、SAN。
温暖度分为适宜、寒冷、炎热、严寒、酷热。

2、计算
温暖度由环境温度、温度抗性计算得到。计算过程不显示，玩家只能看到温暖度。

- 其中环境温度由天气、季节确定，=季节基准温度+天气影响值+场景影响值，取值为摄氏度。
- 其中适宜温度由装备、道具、环境影响，是一个范围，无装备、道具、环境影响时，适宜温度为10-25摄氏度，低温适宜值为10，高温适宜值为25.
- 环境温度大于等于低温适宜值时，小于等于高温适宜值，温暖度为适宜。
- 环境温度小于低温适宜值时，且差值不大于10，温暖度为寒冷。
- 环境温度大于高温适宜值时，且差值不大于10，温暖度为炎热。
- 环境温度小于低温适宜值时，且差值大于10，温暖度为严寒。
- 环境温度大于高温适宜值时，且差值大于10，温暖度为酷热。

#### 负重

1、作用
玩家的负重是指玩家当前携带的物品重量，可以超过最大负重，但是超过最大负重后，玩家无法进行移动交互。
负重以千克为单位，以浮点数表示。且保留一位小数。
负重率是指玩家当前负重与最大负重的比例，负重的各种影响以负重率计算。

2、最大负重
最大负重是指玩家可以携带并移动的最大物品重量，受力量值影响。
计算公式为：最大负重=力量\*5+负重修正。
负重修正默认为0，可由装备、道具、状态、事件等影响。

3、负重率
负重率 = 当前负重/最大负重
不同的负重率会导致不同的影响，具体影响按具体机制中描述进行。

### 基础属性

基础属性、及经验（如果有的话）的变动，除特别说明外，均含以下方式：

- 事件
- 食物
- 道具

#### 力量

力量描述为玩家的力量大小，影响玩家的负重能力、攻击伤害，以及某些交互需要力量判定。

1、取值
力量取值为整数，取值范围为0-100。
力量初始值为10，实际力量=力量+力量修正值，可由装备、道具、状态、事件等影响获得临时力量修正，修正后的值可以超出取值范围。
力量可以成长。

2、成长
力量的成长或者变动有以下方式：

- 以近战方式攻击敌人
- 在50%负重以上状态进行活动

力量通过累计力量经验值的方式进行成长，升级所需经验=当前力量等级\*100。力量100后，不再成长，且不再获得力量经验。
各方式获得的经验值在对应条目中描述。

#### 敏捷

敏捷是指玩家的反应能力，例如移动速度、闪避概率等，以及某些交互需要敏捷判定。

1、作用
敏捷取值为整数，取值范围为0-100。
初始值为10，实际敏捷=敏捷+敏捷修正值。修正后的值可以超出取值范围。
敏捷可以成长。

2、成长
敏捷的成长或者变动有以下方式：

- 闪避攻击
- 在50%负重以下状态进行活动

敏捷通过累计敏捷经验值的方式进行成长，升级所需经验=当前敏捷等级\*100。敏捷100后，不再成长，且不再获得敏捷经验。
各方式获得的经验值在对应条目中描述。

#### 智力

智力是指玩家的智力大小，影响玩家的学习能力、制作能力、SAN值变动，以及某些交互需要智力判定。

1、作用
智力取值为整数，取值范围为0-100。
初始值为10，实际智力=智力+智力修正值。修正后的值可以超出取值范围。
智力可以成长。

2、成长
智力的成长或者变动有以下方式：

- 特定交互

智力通过累计智力经验值的方式进行成长，升级所需经验=当前智力等级\*100。智力100后，不再成长，且不再获得智力经验。
各方式获得的经验值在对应条目中描述。

#### 体质

体质是指玩家的体质大小，影响玩家的HP上限，以及某些交互需要体质判定。

1、作用
体质取值为整数，取值范围为0-100。
初始值为10，实际体质=体质+体质修正值。修正后的值可以超出取值范围。
体质可以成长。

2、成长
体质的成长或者变动有以下方式：

- 特定交互
- 规律作息，规律饮食，体质将逐渐提升

体质通过累计体质经验值的方式进行成长，升级所需经验=当前体质等级\*100。体质100后，不再成长，且不再获得体质经验。
各方式获得的经验值在对应条目中描述。

### 武器熟练度

武器熟练度指玩家使用武器的熟练程度，武器熟练度按照武器类别，例如木剑、铜剑都属于剑。
取值为0-10，无任何修正。
取值为整数，初始值为0。

1、作用
武器熟练度影响玩家使用对应武器技能的伤害及命中、暴击、暴击倍率。
随着武器熟练度提升而解锁新的武器技能。
武器技能及解锁武器技能的等级见战斗机制。

2、成长
武器熟练度的成长：

- 特定事件
- 特定食物
- 特定交互
- 使用该类别武器技能

升级所需经验值 = 当前武器熟练度等级\*100，武器熟练度100后，不再成长，且不再获得武器熟练度经验。
各方式获得的经验值在对应条目中描述。

### 防御属性

防御属性指玩家防御敌人的能力，是0-100之间的整数。
按照类型分为：

- 刺砍防御
- 钝击防御
- 远程防御
- 毒素防御
- 火焰防御

按照百分比降低收到的伤害。

#### 物理防御

物理防御是指玩家防御物理攻击的能力，例如刀剑、钝器等。
取值为0-10，无任何修正。
取值为整数，初始值为0。

1、作用
物理防御影响玩家受到物理攻击的伤害，具体公式见战斗机制。

2、成长
物理防御的成长：

- 特定事件
- 特定食物
- 特定交互
- 使用该类别武器技能

### 技能属性

技能是指玩家的技艺，分为战斗技能和生存技能。

#### 生存技能

生存技能是指玩家在非战斗状态下进行的技能，生命技能影响对应的交互效率、结果、及作为是否可以交互的判定条件。
生存技能取值为0-10，实际生成技能=生存技能+生存技能修正值。修正后的值可以超出取值范围。
生存技能取值为整数。
生存技能可以成长，成长方式与基础属性相同。
生存技能获得经验值的方式主要包括：

- 特定事件
- 特定食物
- 与该生存技能相关的交互

升级所需经验值 = 当前生存技能等级\*100，生存技能满级后，不再成长，且不再获得生存技能经验。

生存技能清单：

- 探索
- 砍伐
- 挖掘
- 采集
- 制作
- 种植
- 烹饪

#### 战斗技能

战斗技能是指玩家战斗中可以使用的技能，详见战斗机制。
战斗技能取值为0-10，无任何修正。
战斗技能取值为整数。
战斗技能可以成长，通过累计经验值获得成长。
战斗技能获得经验值的方式主要包括：

- 特定事件
- 特定食物
- 在战斗中使用该战斗技能

战斗技能的级别，影响该技能的伤害、命中、暴击、暴击倍率等。

战斗技能的解锁来自于：

- 武器熟练度提升
- 特殊事件
- 装备着特定武器

普攻也视为战斗技能，在武器熟练度0时即解锁（即默认解锁）。

#### 被动技能

通过事件、道具习得被动技能。
被动技能在习得时生效，且一直生效，不会随着时间流逝而消失。
被动技能影响的范围与【#### 其他临时状态】相同，数值和效果在习得时确定。
但不显示在状态栏，而是被动技能中。

### (异常)状态

状态指玩家当前处于的额外状态，状态可以由事件、食物、道具、交互及属性的变化获得，持续时间在获得状态时确定，由属性变化获得的状态，在属性恢复后自动取消。（即每次交互、回合刷新一次）

#### 属性变动引发的常驻状态

1、饥饿
饱食度归零时，玩家处于饥饿状态。

- 生命值每小时损失量 = 5% \* 最大生命值，按分钟计。
- SAN值每小时损失量 = 1，按分钟计。
- 体力恢复系数 - 0.5

2、寒冷、炎热、严寒、酷热
玩家当前温暖度处于寒冷、炎热、严寒、酷热时，玩家处于对应状态。
其中寒冷、炎热：

- 生命值每小时损失量 = 5% \* 最大生命值，按分钟计。
- SAN值每小时损失量 = 1，按分钟计。
- 体力恢复系数 - 0.2

其中严寒、酷热：

- 生命值每小时损失量 = 10% \* 最大生命值，按分钟计。
- SAN值每小时损失量 = 1，按分钟计。
- 体力恢复系数 - 0.5

3、负重
根据玩家负重率获得不同状态：
轻盈（<20%）:

- 体力恢复系数 + 0.3
  超重（>80%）:
- 体力恢复系数 - 0.1
  超载（>10%）:
- 体力恢复系数 - 0.3
- 体力消耗系数 + 0.5

#### 其他临时状态

临时状态在施加给角色时确定持续时间/回合，在持续时间/回合结束后自动消失。
使用特定道具、食物可消除对应的临时状态，数值和效果在施加时确定。
临时状态所可能影响的属性包括：

- 生命值恢复速率系数
- 生命值每小时损失/恢复（按分钟计算）
- 生命值每回合损失/恢复
- 饱食度上限系数
- 饱食度自然损失系数
- 体力消耗系数
- 体力恢复系数
- 体力恢复修正
- SAN修正指数
- SAN每小时变动
- SAN每回合变动
- 适宜温度（低温适宜、高温适宜分别）修正值
- 负重修正
- 力量修正
- 敏捷修正
- 智力修正

#### 其他常驻状态

常驻状态没有持续时间，只要条件满足就一直生效。
所能影响的属性与临时状态相同，数值和效果在施加时确认。

## 世界机制

世界随时间而逐渐变化，天气随机变动，昼夜循环，季节更迭。

世界会因玩家的操作和状态而发生改变。

### 环境状态

世界或者环境主要由以下状态值来衡量：
天气、能见度、温度、昼夜、季节、腐化度。

### 天气

天气包括，晴天、多云、阴天、雨天/雪天、暴雨/暴雪、雾/沙尘、雷暴、酸雨。
天气在每日晚上0点变动，持续到次日0点。

- 晴天：温度+5，能见度10。
- 多云：温度+3，能见度9。
- 阴天：温度0，能见度8。
- 雨天/雪天/酸雨：温度-2，能见度6。环境温度大于0时为雨天，否则为雪天。腐化度%的概率将为酸雨。
- 暴雨/暴雪/血雨：温度-5，能见度4。 环境温度大于0时为暴雨，否则为暴雪。腐化度%的概率将为血雨。
- 沙尘/雾：温度0，能见度0。季节为夏、秋季时为雾，否则为沙尘。
- 雷暴：温度-5，能见度4。

天气之间的转换按照以下概率进行
|当前天气|晴天|多云|阴天|雨天/雪天|暴雨/暴雪|沙/雾|雷暴|
|:--|:--|:--|:--|:--|:--|:--|:--|
|晴天|0.5|0.3|0.15|0.03|0.01|0.01|0|
|多云|0.3|0.3|0.3|0.05|0.04|0.01|0|
|阴天|0.2|0.3|0.2|0.2|0.05|0.05|0|
|雨天/雪天|0.4|0.2|0.1|0.1|0.1|0|0.1|
|暴雨/暴雪|0.2|0.2|0.1|0.2|0.1|0|0.2|
|沙/雾|0.25|0.2|0.1|0.1|0.05|0.3|0|
|雷暴|0.4|0.1|0.1|0.1|0.1|0|0.2|

### 能见度

暂时还没想好怎么用。

### 温度

指环境温度，计算方法：
环境温度 = 季节基准温度 + 天气修正 + 地点修正 + 其他修正

### 季节

玩家可见的季节分为春夏秋冬。
每个季节30天。其中前10天为初，中间10天为盛，后10天为末。
季节基准温度：

- 初春：5
- 春季：15
- 春末：20
- 夏初：20
- 夏季：30
- 夏末：35
- 秋初：25
- 秋季：15
- 秋末：5
- 冬初：-5
- 冬季：-20
- 冬末：0

### 昼夜

24小时一个循环。
其中23-1为深夜，2-4为凌晨，5-7为清晨，8-12为上午，13-17为下午，18-20为傍晚，20-22为夜晚。
深夜、凌晨：

- 睡眠可获得2倍效果恢复加成
- 体力消耗系数 + 0.5

### 腐化度

腐化度是环境腐化程度，玩家只能通过特定事件来变动腐化度。
腐化度取值为0-100，初始值为40。
腐化度影响大概包括：

- 敌人的出现状态和频次
- 探索出现对应事件的概率
- 种植、钓鱼等交互的结果

具体在对应事件和事项中标注。

## 战斗机制

适用于任何战斗。
由玩家和敌人的敏捷值决定每回合的出手顺序，玩家每行动一次为一回合。

### 玩家回合

玩家每回合可以进行的操作包括：

- 战斗技能
- 防守
- 使用物品
- 观察/分析
- 逃跑

其中，战斗技能的单独配置，需包括：

- 技能名称
- 技能id
- 武器类型（对于不限制武器的技能则为空）
- 解锁条件（解锁等级、特定武器装备时、事件）
- 技能等级
- 伤害类型
- 伤害（可与力量、武器熟练度、技能等级、武器属性进行计算）
- 命中（可与敏捷、武器熟练度、技能等级、武器属性进行计算）
- 暴击（可与敏捷、武器熟练度、技能等级、武器属性进行计算）
- 暴击倍率（可与敏捷、武器熟练度、技能等级、武器属性进行计算）
- 消耗体力（可与武器熟练度、技能等级、武器属性进行计算）
- 对象
- CD（回合数）
- 施加效果
- 持续时间

其中，防守指进行防守准备，提升下一次敌人行动时降低受到的伤害。

- 消耗体力30，提升下一次敌人行动时降低受到的伤害50%。

其中使用物品：

- 消耗体力0，根据物品类型和物品等级进行效果计算。

其中观察/分析：

- 消耗体力固定10，根据玩家智力属性，概率获得敌人信息。

其中逃跑：

- 消耗体力固定20，逃跑概率=玩家敏捷/（玩家敏捷+敌人敏捷），初见敌人时，获得两倍逃跑概率。特殊战斗禁止逃跑。

### 敌人回合

敌人按照条件和概率行动。
在敌人配置中，应该包括以下内容：

- 敌人名称
- 敌人id
- 敌人类型
- 敌人描述
- 敌人生命值
- 敌人力量
- 敌人敏捷
- 敌人技能清单
- 敌人防御
- 战利品清单及对应概率

其中敌人技能以配置逻辑为主：

- 技能名称
- 技能id
- 使用优先级（优先使用高优先级，优先级相同时按照概率）
- 使用概率
- 使用条件（只有满足条件才会使用）
- 使用次数（一场战斗使用上限）

敌人技能配置：

- 技能名称
- 技能id
- 伤害类型
- 伤害（可与敌人攻击力、技能等级、敌人属性进行计算）
- 命中（可与敌人敏捷、技能等级、敌人属性进行计算）
- 施加效果
- 持续时间
- 对象
- 发动时间（例如蓄力攻击，第三回合生效，0即当场生效）
- CD

### 战斗结束

以某方生命值归零作为战斗结束条件。
1、我方胜利
获得敌方战利品。

2、敌人胜利
游戏结束。进行结算界面。
如果触发战斗的事件中，有配置战败的子事件，则优先触发对应事件。

## 场景与事件

玩家总是身处某个场景之中，例如森林、草地、沙漠、城市、废弃工厂等。

### 场景

场景包括以下配置：

- 场景名称
- 场景id
- 场景描述/事件入口
- 场景图片
- 场景温度影响（-50-50）
- 场景特有的交互内容（探索、人物、游泳、建造营地/进入营地等）

关于场景描述/事件入口的说明

- 场景描述/事件入口：这是显示在场景中主要的文字描述，如果是事件入口则显示为蓝色下划线，玩家点击后进入对应事件。
- 场景描述应该有多种，有优先级，有显示条件（包括但不限于玩家是否第一次造访、玩家的状态、世界腐化度、某些事件是否触发）
- 场景描述/事件入口，可以设置成被动触发，即满足条件且概率抽中此条时，直接进入事件而不显示文字描述。

### 子场景

子场景是场景的子集，例如森林中的小溪、草地上的小屋、沙漠中的洞穴等。
子场景的配置与场景类似，但子场景没有温度影响，且子场景的交互内容一般与场景不同。
子场景包括以下配置：

- 场景名称
- 场景id
- 场景描述/事件入口
- 场景图片
- 场景温度影响（-50-50）（与母场景叠加）
- 是否地牢场景
- 子场景特有的交互内容
- 地牢场景特有内容：
  - 相邻子场景（只有地牢子场景生效）
  - 方位移动交互（根据相邻子场景，显示东西南北直接移动子场景）

子场景中不会触发母场景的事件、不显示母场景的交互，但是温度影响相互叠加。
子场景图片、描述等为空时显示母场景的。

### 交互内容

交互内容的配置：

- 交互名称
- 交互id
- 交互类型（探索、事件、功能、进入子场景）
- 交互条件（显示此交互的条件，例如沙滩场景，只有触发过“发现海怪”事件后，才会显示“海怪”选项，例如只有晚上1点才有海民选项。）
- 交互描述（显示在交互按钮上）
- 消耗的资源（体力、时间等）

关于交互类型的说明：

- 探索：探索场景，刷新场景描述/事件入口
- 事件：直接触发事件，不刷新场景描述/事件入口
- 功能：直接进入功能界面，例如建造、种植、制作等
- 进入/切换子场景：直接进入子场景，不刷新当前场景描述/事件入口

### 事件

事件根据玩家交互、或被动触发。
事件配置包括以下内容：

- 事件名称
- 事件备注（不会显示给玩家）
- 事件描述（显示在上面的文字）
- 事件图片

- 进入子事件
- 子事件配置
  - 子事件名称
  - 文本描述（可根据条件显示不同的描述）
  - 图片描述（如果有的话，覆盖事件图片）
  - 选项列表
    - 选项
    - 选项描述
    - 选项可用条件
    - 选项结果
      - 选项类型（结束、战斗、交易、进入CG或其他可能的，留空则无额外操作进入下一个子事件）
      - 参数（战斗的敌人、胜利的子事件、战败的子事件、CG的id等）
      - 下一个子事件
      - 属性值、物品、技能等及其他标志位变动（概率、条件分别）

举例：
渔民事件：
描述渔民正在钓鱼，选项【搭话】【攻击】
【搭话】询问海怪事件，渔民告诉你海怪只在晚上1点出现，赠与你力量药水【感谢离开】结束事件。
【攻击】进入战斗，战斗胜利获得金币、鱼、海怪蛋等，进入【击杀】子事件，【拾取物品】获得服装，结束事件。


## 基地

玩家可以在有限的地点（离谱海滩等平坦地点）建立基地。基地是玩家活动的大本营，在基地可以储存物品、睡觉、制作、建造等。
敌人可能会定期袭击玩家基地。（触发事件）
建造可以建造防御建筑、生产建筑（田地、集雨器）、储存建筑、生活建筑（例如床屋子火堆等）、制作建筑（工作台、铁砧等）等。
主基地只能有一个。在其他地点勘探完毕后，玩家可以选择迁移基地。
迁移基地需要大量时间和材料，并触发事件。

### 临时基地

玩家可以消耗某些资源就地扎营，不限制地点，除了地牢。临时营地也可以睡觉、或使用火堆烹饪。

# 背景

背景基于克苏鲁+SCP的世界观。

在太平洋的中间，有一座无名的岛屿。这座岛本身是正常的热带岛屿，但在五年前，一场海底地震震裂了海沟里的古老沉积层，释放出一种“远古孢子”（The Spores）。孢子并非来自于地球本身，而是早在地球生命诞生之前就降临在地球上的宇宙造物。

孢子核心会无意识地在一定范围内散发能量场，导致生物发生变异。变异后的生物体内会产生新的污染源，即便能量场不存在，接触污染源也会让生物继续变异。

SCP基金会介入后，对孢子本身进行收容，但物质场无法收容。在一段时间的研究后，随着工作人员的陆续变异，最终SCP基金会决定将整个岛一起收容。整个岛屿处于所有物质只进不出的状态。主角因此来到这里。

因此岛上的生物因此变得极具攻击性且外形扭曲，但它们依然是物理存在，可以被长矛刺穿，被子弹打死。

变异的影响分为身体与大脑。身体的变异包括但不限于形态的变化，额外的肢体、超越正常人类的能力。
大脑的变化主要分三种，一种是宗教般的信仰狂热，一种是彻底的疯狂只保留进食冲动，一种是冷漠拒绝理解。

通关结局：
1、修复之前研究基地遗留的岛屿中心最高处的信号塔以呼叫救援。
2、造一个好大船只离开。
保护特定NPC以解锁浪漫结局。
在感染状态离开，解锁0号病人结局。
发现真相，解锁真相结局。
3、低SAN值下，打倒变异人首领，解锁融入结局。
4、发现真相，并通过引爆彻底解决孢子，解锁寂静归零结局。
5、4在自身不死亡条件下，解锁真结局，英雄。

# 游戏核心机制

## 场景、事件、交互
场景、事件、交互是游戏的核心。

玩家总是在场景中，场景包括固定的交互和事件入口。

通过交互和事件入口，玩家进入事件中。

通过触发事件，以及在事件中的交互，进入战斗、建造、CG等界面、改变玩家的状态、物品、属性等。

## 事件触发与时间
对于buff、日期变化等时间判定，以动作完成后时间进行判定。
例如有一个每分钟扣10血的buff，玩家选择探索（耗时3小时）。
无需计算中间时间，在3小时结束后，统一处理所有与时间相关的事件。（扣除血量）
天气、季节、日期等时间变化，也会在动作完成后时间进行判定。



## 界面与UI

界面与UI是游戏的基础，玩家通过界面与UI进行交互，了解游戏状态，进行游戏操作。

### 启动界面

游戏启动后的主菜单界面，包括以下内容：

- 游戏名称
- 开始新游戏
- 继续游戏
- 设置
- 退出游戏

### CG界面

CG界面是游戏中的过场动画，开局、结局及重要场景应使用CG界面。

CG界面不显示血条等其他UI，仅显示文字描述和图片。

和玩家的选项。

根据玩家的选项，选择进入下一个CG界面或进入其他界面。

### 常态界面

游戏的常态界面，显示以下内容：

- 最上：第几日、季节、天气、时间（仅拥有手表或其他时间道具后）以及时间段的icon
- 次上：生命值、饱食度、体力、SAN（仅有特定道具时显示数值进度条，否则不显示SAN：\*\*，而是显示文本，例如理智、混乱）
- 次次上：左侧设置、背包、属性操作按钮，右侧当前场景名称（沙滩、沙滩-小屋、沙滩-洞穴-洞穴2）
- 中间：当前场景描述、可交互内容以蓝色下划线显示，点击后进入对应事件或交互，特别内容可以以不同颜色大小字体显示
- 下方：玩家主交互区域
  - 当前场景特有的交互按钮，例如探索、劈柴、建造等

其中中间主要描述、下方交互按钮的数据主要来自于scene。
交互按钮会转向事件。
交易界面同样在事件总点击交互再进入，不直接从场景进入。
在基地中，特有的已完成建筑交互按钮，转向特有的建筑界面。（数据来自于build）在建筑界面中的交互按钮，再转向对应的界面，例如烹饪、制作等。
基地中特有的建造按钮，直接转向建造页面。

### 事件界面

进入事件后的显示内容：

- 最上：第几日、季节、天气、时间（仅拥有手表或其他时间道具后）以及时间段的icon
- 次上：生命值、饱食度、体力、SAN（仅有特定道具时显示数值进度条，否则不显示SAN：\*\*，而是显示文本，例如理智、混乱）、其他（例如超重时显示超重icon，寒冷时显示寒冷icon）
- 次次上：左侧设置、背包、属性操作按钮均隐藏，右侧当前场景名称（沙滩、沙滩-小屋、沙滩-洞穴-洞穴2）
- 中间：当前事件描述，特别内容可以以不同颜色大小字体显示
- 下方：当前事件特有的交互选项

其中中间主要描述、下方交互按钮的数据主要来自于event。

### 建筑页面

建筑页面显示玩家当前选择的建筑。

- 最上：第几日、季节、天气、时间（仅拥有手表或其他时间道具后）以及时间段的icon
- 次上：生命值、饱食度、体力、SAN（仅有特定道具时显示数值进度条，否则不显示SAN：\*\*，而是显示文本，例如理智、混乱）、其他（例如超重时显示超重icon，寒冷时显示寒冷icon）

- 中间：建筑描述。
- 下方：交互按钮，包括对应的功能、拆除、升级等。其中功能按钮转向对应的界面。


### 战斗界面

进入战斗后的显示内容：

- 最上：第几日、季节、天气、时间（仅拥有手表或其他时间道具后）以及时间段的icon
- 次上：玩家生命值、体力、SAN
- 次次上：敌人生命值（敌人可能有多个）
- 中间：战斗描述，特别内容可以以不同颜色大小字体显示
- 下方：玩家战斗战斗交互选项

- 次上：生命值、饱食度、体力、SAN（仅有特定道具时显示数值进度条，否则不显示SAN：\*\*，而是显示文本，例如理智、混乱）、其他（例如超重时显示超重icon，寒冷时显示寒冷icon）

import type { Condition } from './effect'

// ============================================================
// 事件配置
// ============================================================

/**
 * 事件配置
 * 事件是游戏内容的核心载体，负责：
 * - 展示叙事文本（含根据条件变化的文本变体）
 * - 提供选项供玩家选择
 * - 通过选项结果驱动：属性变动、物品获得、战斗触发、CG播放、场景切换、标志位设置等
 *
 * 事件由多帧（EventFrame）组成，每帧可包含多个选项。
 * 事件总是以某个结束型选项终止（结束事件、进入战斗、切换场景、播放CG等）。
 */
export interface GameEvent {
  /** 事件唯一ID */
  id: string
  /** 事件名称（开发者可见） */
  name: string
  /** 事件备注 */
  notes?: string

  /** 事件帧序列（至少一帧） */
  frames: EventFrame[]

  /** 事件触发条件（不满足则无法进入此事件） */
  triggerCondition?: Condition

  /** 事件开始时自动触发的效果（进入事件时立即执行） */
  onEnterEffects?: EffectResult[]

  /** 事件类型 */
  eventType: EventType

  /** 事件是否可重复触发（false表示只能触发一次） */
  isRepeatable: boolean
  /** 触发后设置的标志位（用于追踪是否已触发过） */
  triggeredFlag?: string

  /** 事件是否可被中断（如战斗中逃跑返回事件） */
  isInterruptible: boolean

  /** 事件图片（覆盖场景图片） */
  backgroundImage?: string
  /** 事件图片变体（根据条件显示不同图片） */
  backgroundImageVariations?: EventImageVariation[]
}

/**
 * 事件类型
 */
export enum EventType {
  /** 普通事件 */
  NORMAL = 'normal',
  /** 战斗事件（可直接进入战斗） */
  BATTLE = 'battle',
  /** 交易事件 */
  TRADE = 'trade',
  /** CG事件（播放CG） */
  CG = 'cg',
  /** 遭遇事件（移动途中触发的随机事件） */
  ENCOUNTER = 'encounter',
  /** 梦境事件（低SAN值时触发） */
  DREAM = 'dream',
  /** 特殊事件（结局、关键剧情等） */
  SPECIAL = 'special',
}

/**
 * 事件图片变体
 */
export interface EventImageVariation {
  /** 图片资源ID */
  imageId: string
  /** 显示条件 */
  condition: Condition
  /** 过渡效果 */
  transitionEffect?: 'fade' | 'glitch' | 'instant'
}

// ============================================================
// 事件帧
// ============================================================

/**
 * 事件帧
 * 事件的每个步骤/页面，包含描述文本和选项列表。
 * 当玩家选择某个选项后，根据选项结果跳转到下一帧或结束事件。
 */
export interface EventFrame {
  /** 帧ID（事件内唯一） */
  id: string
  /** 帧序号（用于排序） */
  order: number

  /** 帧文本（展示给玩家的内容，支持换行） */
  text: string
  /** 文本变体（根据条件显示不同文本，如SAN值影响叙事） */
  textVariations?: EventTextVariation[]

  /** 帧图片（可选，覆盖事件的背景图） */
  imageId?: string

  /** 帧选项列表 */
  options: EventOption[]

  /** 帧显示条件（满足条件时才显示此帧，否则跳过） */
  displayCondition?: Condition

  /** 进入此帧时自动触发的效果 */
  onEnterEffects?: EffectResult[]
  /** 离开此帧时自动触发的效果 */
  onExitEffects?: EffectResult[]
}

/**
 * 事件文本变体
 */
export interface EventTextVariation {
  /** 变体文本 */
  content: string
  /** 显示条件 */
  condition: Condition
}

// ============================================================
// 事件选项
// ============================================================

/**
 * 事件选项
 * 玩家在事件帧中可点击的选项按钮。
 * 选项可以：
 * - 跳转到同一事件的另一个帧（继续叙事）
 * - 结束事件（返回场景）
 * - 触发战斗
 * - 播放CG
 * - 进入交易界面
 * - 切换场景
 * - 触发其他事件
 */
export interface EventOption {
  /** 选项ID（帧内唯一） */
  id: string

  /** 选项文本（显示在按钮上） */
  text: string
  /** 文本变体（根据条件显示不同文本） */
  textVariations?: EventTextVariation[]

  /** 选项描述（可选，长按或tooltip显示） */
  description?: string

  /** 选项显示条件（满足条件时才显示此选项） */
  displayCondition?: Condition
  /** 选项可用条件（满足条件时才可点击，不满足时灰显） */
  availableCondition?: Condition
  /** 不可用时的提示文本 */
  unavailableTooltip?: string

  /** 选项样式 */
  optionStyle?: EventOptionStyle

  /** 选择此选项消耗的资源 */
  costs?: EventOptionCost[]

  /** 选项结果（选择后发生的事情） */
  result: EventOptionResult

  /** 选项优先级（数字越大显示越靠前） */
  displayPriority?: number

  /** 是否需要确认弹窗（防止误操作重要选项） */
  requiresConfirmation?: boolean
  /** 确认弹窗文本 */
  confirmationText?: string

  /** 此选项是否只能选择一次（再次进入此帧时灰显或隐藏） */
  isOneTime?: boolean
  /** 选择后设置的标志位 */
  selectedFlag?: string

  /** 选择后选项文本的变化（如"敲门"→"再次敲门"） */
  textAfterSelected?: string
}

/**
 * 事件选项样式
 */
export enum EventOptionStyle {
  /** 默认样式 */
  DEFAULT = 'default',
  /** 危险操作（红色，如攻击、跳崖等） */
  DANGER = 'danger',
  /** 特殊/关键操作（金色/高亮） */
  SPECIAL = 'special',
  /** 隐藏选项（满足条件才显示，显示时可能是半透明或特殊效果） */
  HIDDEN = 'hidden',
  /** 理智相关选项（低SAN值下特有样式） */
  MADNESS = 'madness',
}

/**
 * 事件选项消耗
 */
export interface EventOptionCost {
  /** 消耗类型 */
  costType: EventOptionCostType
  /** 消耗值 */
  value: number
  /** 消耗的物品ID（costType为ITEM时使用） */
  itemId?: string
  /** 消耗的物品数量（costType为ITEM时使用） */
  itemQuantity?: number
}

/**
 * 事件选项消耗类型
 */
export enum EventOptionCostType {
  STAMINA = 'stamina',
  SATIETY = 'satiety',
  SAN = 'san',
  HP = 'hp',
  ITEM = 'item',
}

// 事件选项结果
// ============================================================

/**
 * 事件选项结果（联合类型）
 * 选项的结果必为以下之一：

 * - 结束事件返回场景
 * - 触发战斗
 * - 播放CG
 * - 进入交易
 * - 切换场景
 * - 触发另一个事件
 */
export type EventOptionResult =
  | NextFrameResult
  | EndEventResult
  | TriggerBattleResult
  | PlayCGResult
  | OpenTradeResult
  | SwitchSceneResult
  | TriggerEventResult

 * 跳转到同一事件的另一个帧

  /** 目标帧ID（同一事件内） */

  /** 执行的效果列表 */

  /** 设置标志位 */
  setFlags?: Record<string, FlagValue>
}

/**
 * 结束事件，返回场景
 */
export interface EndEventResult {
  type: 'endEvent'
  /** 执行的效果列表 */
  effects?: EffectResult[]
  /** 设置标志位 */
  setFlags?: Record<string, FlagValue>
  /** 返回场景后显示的文字（可选，如"你离开了小屋"） */

}

/**

export interface TriggerBattleResult {

  /** 战斗胜利后跳转的帧ID（同一事件内） */
  victoryFrameId?: string
  /** 战斗失败后跳转的帧ID（不填则战斗失败进入结局判定） */
  defeatFrameId?: string
  /** 战斗逃跑后跳转的帧ID（不填则逃跑后返回场景） */
  escapeFrameId?: string
  /** 是否可逃跑 */
  canEscape?: boolean
  /** 战前执行的效果列表 */
  effects?: EffectResult[]
  /** 设置标志位 */
  setFlags?: Record<string, FlagValue>
  /** 是否有初见加成（初见时逃跑概率翻倍） */
  firstEncounterBonus?: boolean
}

/**
 * 播放CG
 */
export interface PlayCGResult {
  type: 'playCG'
  /** CG配置ID */
  cgId: string
  /** CG结束后跳转的帧ID（不填则CG结束后返回场景） */
  returnFrameId?: string
  /** 执行的效果列表 */

  /** 设置标志位 */
  setFlags?: Record<string, FlagValue>
}

/**
 * 切换场景
 */
export interface SwitchSceneResult {
  type: 'switchScene'
  /** 目标场景ID */
  sceneId: string
  /** 目标子场景ID（可选） */
  subSceneId?: string
  /** 执行的效果列表 */
  effects?: EffectResult[]
  /** 设置标志位 */
  setFlags?: Record<string, FlagValue>
  /** 进入新场景后显示的文字（可选） */
  enterText?: string
}

/**
 * 触发另一个事件

export interface TriggerEventResult {

  /** 目标事件结束后跳转的帧ID（不填则返回场景） */

  /** 执行的效果列表 */

  /** 设置标志位 */

/**
 * 标志位配置
 * 标志位用于追踪游戏中的各种状态：
 * - 事件是否已触发
 * - 场景描述是否已看过
 * - 物品是否已获得过
 * - 任务进度
 * - 各种解锁状态
 * 等等

  /** 标志位唯一ID */
  id: string
  /** 标志位名称（开发者可见，便于调试） */
  name: string
  /** 标志位备注 */
  notes?: string

  /** 标志位数据类型 */
  type: FlagType

  /** 默认值 */
  defaultValue: FlagValue
}

  /** 数值型（如"击杀某敌人次数"、"捐赠物资数量"） */
  NUMBER = 'number',
  /** 字符串型（如"当前任务阶段名称"、"NPC好感度等级"） */
  STRING = 'string',
}

  /** 布尔取反 */
  TOGGLE = 'toggle',
  /** 数值增加 */
  ADD = 'add',
  /** 数值减少 */
  SUBTRACT = 'subtract',
}

}

// 场景配置
// ============================================================

/**
 * 场景配置
 * 场景是游戏世界的核心容器，负责：
 * 1. 展示环境描述（文字+图片）
 * 2. 提供事件入口（可点击的蓝色链接或自动触发的事件）
 * 3. 提供固定的交互按钮（探索、进入子场景、建造、休息等功能）
 * 场景本身不直接触发效果、战斗、采集资源等，
 * 这些逻辑统一通过"进入事件"来实现，由事件系统处理。
 */
export interface Scene extends BaseScene {
  // 该场景是否可以作为基地建造地点
  canBuildBase: boolean

  // 该场景包含的子场景ID列表（与 SubScene.parentSceneId 形成双向关联）
  subSceneIds?: string[]
}

// ============================================================
// 子场景配置
// ============================================================

/**
 * 子场景配置
 * 子场景是场景的子集，例如森林中的小溪、沙滩上的洞穴等。
 * 子场景不会触发母场景的descriptions，不显示母场景的interactions，
 * 但温度影响与母场景叠加。
 * 图片、描述等为空时使用母场景的对应内容。
 */
export interface SubScene extends BaseScene {
  // 母场景ID
  parentSceneId: string

  // ========== 地牢场景特有 ==========
  // 相邻子场景映射（方向 -> 相邻子场景ID）
  // 非地牢场景忽略此字段
  adjacentSubScenes?: {
    north?: string
    south?: string
    east?: string
    west?: string
    up?: string
    down?: string
  }
}

// ============================================================
// 场景描述
// ============================================================

/**
 * 场景描述条目
 * 场景中显示的文字内容，可以包含多个事件入口。
 * 多个描述条目根据优先级和显示条件竞争展示。
 */
export interface SceneDescription {
  // 描述ID
  id: string
  // 优先级（数字越大优先级越高，同优先级时随机选取一条展示）
  priority: number
  // 显示条件（满足条件时此描述才可能被选中展示）
  displayCondition?: Condition

  // 文本内容（展示给玩家的主文本）
  // 使用 {eventKey} 占位符标记事件入口位置，例如：
  // "你看到{strange_altar}矗立在林间空地中央，旁边{footprints}通向密林深处。"
  // {strange_altar} 和 {footprints} 会被替换为蓝色下划线文本，对应 eventEntries 中的 key
  text: string
  // 文本变体（根据条件显示不同文本，例如根据SAN值改变描述风格）
  textVariations?: SceneTextVariation[]

  // 文本样式（特殊效果，受SAN值等影响）
  style?: SceneTextStyle

  // ========== 事件入口配置 ==========
  // 文本中可点击的事件入口列表
  // key 对应 text 中的 {key} 占位符
  eventEntries?: SceneEventEntry[]

  // ========== 自动触发配置 ==========
  // 是否可自动触发（满足条件时自动进入某个事件）
  isAutoTrigger: boolean
  // 自动触发对应的事件入口 key（若不填则取 eventEntries 中的第一个）
  autoTriggerEventKey?: string
  // 自动触发的概率（0-1，1为必定触发，仅在isAutoTrigger为true时有效）
  autoTriggerProbability?: number
  // 自动触发是否需要先展示描述（若为true，则先展示描述再自动进入事件；false则跳过描述直接进入事件）
  showDescriptionBeforeAutoTrigger?: boolean
  // 自动触发延迟（毫秒，showDescriptionBeforeAutoTrigger为true时，展示描述后延迟多久自动进入事件）
  autoTriggerDelayMs?: number

  // 是否在任意事件入口触发后清除此描述（通过设置seenFlag实现）
  removeAfterInteraction?: boolean

  // ========== 展示频率控制 ==========
  // 此描述是否只能被看到一次
  isOneTime: boolean
  // 看过此描述后设置的标志位（用于后续判定是否已看过）
  seenFlag?: string
  // 此描述可被看到的次数上限（-1表示无限次，配合seenFlag使用）
  viewLimit?: number

  // ========== 环境限制 ==========
  // 是否在特定时间段显示
  timeOfDayRestriction?: TimeOfDay[]
  // 是否在特定天气显示
  weatherRestriction?: WeatherType[]
  // 是否在特定季节阶段显示
  seasonRestriction?: SeasonPhase[]
  // 显示所需的腐化度范围
  corruptionRange?: {
    min: number
    max: number
  }
}

/**
 * 场景事件入口
 * 描述文本中的可点击蓝色链接，点击后进入对应事件
 */
export interface SceneEventEntry {
  // 入口唯一标识（对应 text 中的占位符 {key}）
  key: string
  // 显示文本（即蓝色下划线文字的内容）
  displayText: string
  // 事件ID（点击后进入此事件）
  eventId: string
  // 显示条件（满足条件时此入口才显示并可点击）
  displayCondition?: Condition
  // 可用条件（满足条件时此入口才可点击，不满足时仅显示文本但不可点击）
  availableCondition?: Condition
  // 不可用时的提示文本
  unavailableTooltip?: string
  // 是否在不满足条件时完全隐藏（不渲染此入口文本）
  hideWhenUnavailable?: boolean
  // 点击后是否移除此入口（下次展示此描述时不再出现）
  removeAfterClick?: boolean
  // 点击后设置的标志位
  clickFlag?: string
  // 点击后的文本变体（如点击前"奇怪的祭坛"，点击后变为"已被调查的祭坛"）
  textAfterClick?: string
}

/**
 * 场景文本变体
 * 根据条件（如SAN值）替换描述文本及事件入口
 */
export interface SceneTextVariation {
  // 变体文本内容（同样支持 {eventKey} 占位符）
  content: string
  // 显示条件
  condition: Condition
  // 变体样式（覆盖默认样式）
  style?: SceneTextStyle
  // 变体对应的事件入口列表（覆盖 SceneDescription 的 eventEntries）
  eventEntries?: SceneEventEntry[]
}

/**
 * 场景文本样式

  /** 技能描述 */

  /** 技能图标资源ID */
  iconId: string

  /** 技能类型 */
  skillType: 'passive'

  /** 被动效果列表（习得后永久生效） */
  effects: EffectResult[]

  /** 属性加成（直接作用于玩家属性面板） */
  attributeBonuses?: SkillAttributeBonus[]

  /** 该被动是否可重复习得（多数被动仅可习得一次） */
  isStackable: boolean
  /** 最大叠层数（isStackable为true时有效） */
  maxStacks?: number

  /** 习得条件（仅用于事件或道具中引用，被动技能本身不主动触发习得） */
  acquireCondition?: Condition
}

/**
 * 异常状态配置
 * 定义一种可施加到玩家或敌人身上的临时状态模板。
 * 状态的实例化（施加到具体目标、带剩余时间）是运行时数据。
 * 施加/移除通过 Effect 系统中的 StatusEffect 触发。
 */
export interface StatusConfig {
  /** 状态唯一ID */
  id: string
  /** 状态名称（显示用） */
  name: string
  /** 状态备注（开发者可见） */
  notes?: string
  /** 状态描述（显示给玩家） */
  description: string
  /** 状态描述变体（根据条件显示不同文本） */
  descriptionVariations?: StatusDescriptionVariation[]

  /** 状态图标资源ID */
  iconId: string

  // ============================================================
  // 状态分类
  // ============================================================

  /** 状态类型 */
  statusType: StatusType

  // ============================================================
  // 时间与叠层
  // ============================================================

  /** 默认持续时间，具体以施加的事件时间为准 */
  defaultDuration: StatusDuration

  /** 叠加规则 */
  stackingRule: StatusStackingRule

  // ============================================================
  // 效果
  // ============================================================

  /** 状态效果（每周期触发） */
  effects: StatusEffectConfig[]

  /** 状态施加时立即触发的效果（仅一次） */
  onApplyEffects?: EffectResult[]
  /** 状态移除时触发的效果（仅一次） */
  onRemoveEffects?: EffectResult[]

  // ============================================================
  // 显示与视听
  // ============================================================

  /** 视觉效果 */
  visualEffects?: StatusVisualEffect[]

  // ============================================================
  // 移除与互斥
  // ============================================================

  /** 可否被驱散（某些状态如诅咒不可被常规手段移除） */
  isDispellable: boolean

  /** 被移除时的提示文本 */
  exclusiveRemovalText?: string

  // ============================================================
  // 死亡相关
  // ============================================================

  /** 此状态是否在战斗结束后移除 */
  removeOnBattleEnd: boolean
  /** 此状态是否在休息/睡觉后移除 */
  removeOnRest: boolean
}

// 状态类型

/**
 * 状态类型
 */
export enum StatusType {
  /** 负面状态（中毒、流血、恐惧等） */
  DEBUFF = 'debuff',
  /** 正面状态（力量增强、护盾、加速等） */
  BUFF = 'buff',
  /** 中立状态（标记类，如"被猎人标记"，本身无害） */
  NEUTRAL = 'neutral',
  /** 特殊状态（诅咒、感染、腐化等，有独特机制） */
  SPECIAL = 'special',
}

// ============================================================
// 时间与叠层
// ============================================================

/**
 * 状态持续时间
 */
export interface StatusDuration {
  /** 持续时间值 */
  value: number
  /** 时间单位 */
  unit: 'turn' | 'minute' | 'hour' | 'permanent'
  /** 最小持续时间限制（施加时若指定时间小于此值则使用此值） */
  minValue?: number
  /** 最大持续时间限制（施加时若指定时间大于此值则使用此值） */
  maxValue?: number
}

/**
 * 状态叠加规则
 */
export enum StatusStackingRule {
  /** 不可叠加：已存在时忽略新施加（不刷新时间） */
  NONE = 'none',
  /** 刷新时间：已存在时仅刷新持续时间，不增加层数 */
  REFRESH = 'refresh',
  /** 独立叠加：每层独立计时，效果×层数 */
  STACK_INDEPENDENT = 'stackIndependent',
  /** 刷新叠加：层数+1且刷新所有层持续时间 */
  STACK_REFRESH = 'stackRefresh',
  /** 仅叠层：层数+1但保持原有持续时间不变 */
  STACK_NO_REFRESH = 'stackNoRefresh',
}

/**
 * 武器类型配置
 * 定义一类武器的通用属性，如剑、斧、弓、长矛等。
 * 具体的武器物品（如"铁剑"、"铜剑"）在 item.ts 中配置，
 * 通过 weaponTypeId 关联到此类型。
 */
export interface WeaponType {
  /** 武器类型唯一ID */
  id: string
  /** 武器类型名称（显示用，如"剑"、"弓"） */
  name: string
  /** 武器类型描述 */
  description: string
  /** 武器类型图标资源ID */
  iconId: string

  /** 主要伤害类型ID */
  primaryDamageTypeId: string

  /** 是否为远程武器 */
  isRanged: boolean

  /**
   * 该武器类型的默认属性模板
   * 创建该类型的具体武器物品时，可作为 WeaponStats 的默认值
   */
  defaultStats: WeaponTypeDefaultStats

  /**
   * 技能解锁表
   * 键为武器熟练度等级，值为该等级解锁的战斗技能ID列表
   * 例如：{ 0: ['basic_slash'], 3: ['power_strike'], 7: ['whirlwind'] }
   * 熟练度0的技能在装备武器时即解锁（普攻等基础技能）
   */
  skillUnlocks: Record<number, string[]>

  /**
   * 熟练度成长配置
   * 该武器类型的熟练度经验获取倍率
   */
  proficiencyGrowth: WeaponProficiencyGrowth
}

/**
 * 武器类型默认属性
 */
export interface WeaponTypeDefaultStats {
  /** 基础伤害 */
  baseDamage: number
  /** 伤害浮动范围 */
  damageVariance: number
  /** 攻击距离 */
  attackRange: number
  /** 基础命中修正 */
  accuracyModifier: number
  /** 基础暴击率修正 */
  criticalChanceModifier: number
  /** 暴击倍率 */
  criticalMultiplier: number
  /** 攻击速度 */
  attackSpeed: number
  /** 每次攻击基础消耗体力 */
  staminaCostPerAttack: number
}

/**
 * 武器熟练度成长配置
 */
export interface WeaponProficiencyGrowth {
  /** 使用该类型武器技能攻击命中时获得的熟练度经验 */
  expPerHit: number
  /** 使用该类型武器技能造成暴击时额外获得的经验 */
  expPerCriticalHit: number
  /** 击杀敌人时额外获得的经验 */
  expPerKill: number
}
